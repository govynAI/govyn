  <name>Task 1: In-memory cost aggregator with rolling time windows</name>
  <files>src/cost-aggregator.ts, src/types.ts, tests/cost-aggregator.test.ts</files>
  <action>
1. Add to src/types.ts:
   - `CostRecord` interface: `{ agentId: string; model: string; provider: ProviderType; inputTokens: number; outputTokens: number; inputCost: number; outputCost: number; totalCost: number; priced: boolean; timestamp: number }`
   - `CostSummary` interface: `{ agentId: string; totalCost: number; inputCost: number; outputCost: number; totalInputTokens: number; totalOutputTokens: number; requestCount: number; models: Record<string, { cost: number; requests: number; inputTokens: number; outputTokens: number }> }`
   - `TimePeriod` type: `'hour' | 'day' | 'month' | 'all'`

2. Create src/cost-aggregator.ts with class `CostAggregator`:
   - Internal storage: `records: CostRecord[]` — flat array of all cost records (in-memory, no persistence for Phase 2).
   - `recordCost(record: CostRecord): void` — appends a CostRecord to the array. This is called after every proxied request.
   - `getSummary(options?: { agentId?: string; period?: TimePeriod }): CostSummary[]`
     - Filters records by agentId if provided.
     - Filters records by time period:
       - 'hour': records from the last 60 minutes
       - 'day' (alias 'today'): records from current calendar day (midnight UTC to now)
       - 'month': records from current calendar month (1st of month UTC to now)
       - 'all' or undefined: all records
     - Groups remaining records by agentId.
     - For each agent group, computes: totalCost, inputCost, outputCost, totalInputTokens, totalOutputTokens, requestCount, and per-model breakdown.
     - Returns array of CostSummary objects.
   - `getModelSummary(options?: { period?: TimePeriod }): Record<string, { cost: number; requests: number; inputTokens: number; outputTokens: number }>`
     - Groups all records by model name, aggregates cost and token totals.
   - `getUnpricedModels(): string[]`
     - Returns list of unique model names where priced === false.
   - `clear(): void` — resets all records (useful for testing and future budget reset).

   - Memory management: For Phase 2 MVP, no eviction. Records accumulate in memory. Phase 3 budget enforcement will add daily/monthly reset logic. Phase 4 logging will persist records. If memory becomes a concern, add a max records limit with LRU eviction (but defer this optimization).

3. Create tests/cost-aggregator.test.ts with tests:
   - recordCost adds a record and getSummary returns it
   - getSummary with agentId filter returns only that agent's records
   - getSummary with period 'day' filters to current day records only
   - getSummary with period 'hour' filters to last 60 minutes only
   - getSummary correctly aggregates multiple records per agent (totalCost sums, requestCount increments)
   - getSummary per-model breakdown shows correct per-model cost and token counts
   - getModelSummary groups by model correctly
   - getUnpricedModels returns models flagged as unpriced
   - clear() resets all records
   - Multiple agents: getSummary without filter returns separate summaries for each agent
   - Records with timestamps in the past are correctly filtered by time period
  </action>

2. Update src/proxy.ts forwardRequest signature and logic:
   - Add parameters: `agentId: string`, `pricingTable: PricingTable`, `aggregator: CostAggregator`
   - Updated signature: `forwardRequest(req, res, routeMatch, agentId, pricingTable, aggregator)`
   - After the upstream response is fully received (non-streaming: pipe ends, streaming: SSE stream ends), extract tokens and record cost. Implementation:

   **Non-streaming path:** Instead of just piping the upstream response, also collect the body chunks for token extraction. Use a listener on the upstream response data events to accumulate body into a buffer, then on 'end', extract tokens, calculate cost, and record. The piping to the client response happens unchanged (concurrent with buffering). This adds minimal overhead because we only parse the JSON after the response is complete.

   ```typescript
   // Accumulate body for token extraction (concurrent with pipe to client)
   const bodyChunks: Buffer[] = [];
   upstreamRes.on('data', (chunk) => bodyChunks.push(chunk));
   upstreamRes.on('end', () => {
     const body = Buffer.concat(bodyChunks).toString('utf8');
     const usage = extractTokenUsage(body, routeMatch.providerType);
     if (usage) {
       const cost = calculateCost(usage, pricingTable);
       aggregator.recordCost({
         agentId,
         model: usage.model,
         provider: routeMatch.providerType,
         inputTokens: usage.inputTokens,
         outputTokens: usage.outputTokens,
         inputCost: cost.inputCost,
         outputCost: cost.outputCost,
         totalCost: cost.totalCost,
         priced: cost.priced,
         timestamp: Date.now(),
       });
     }
   });
   ```

   **SSE streaming path:** Collect SSE data chunks as strings. After the stream ends, call extractTokenUsageFromSSE to parse token usage from the accumulated chunks. Calculate cost and record. The SSE piping to the client is unchanged.

   ```typescript
   // Accumulate SSE chunks for token extraction
   const sseChunks: string[] = [];
   upstreamRes.on('data', (chunk) => sseChunks.push(chunk.toString('utf8')));
   // After stream ends (listen on res 'finish' or upstreamRes 'end'):
   const usage = extractTokenUsageFromSSE(sseChunks, routeMatch.providerType);
   if (usage) {
     const cost = calculateCost(usage, pricingTable);
     aggregator.recordCost({ ... });
   }
   ```

   - IMPORTANT: Token extraction and cost recording happen AFTER the response is complete. They are non-blocking relative to the client response — the pipe to the client has already delivered all data. This satisfies the latency requirement.
   - IMPORTANT: For upstream errors (4xx, 5xx), still attempt token extraction from the body (some error responses include usage data). If extraction fails, that is fine — no cost is recorded.
   - Add a console.log for each cost recording: `[govyn] Cost: agent=${agentId} model=${model} tokens=${total} cost=$${totalCost.toFixed(6)} priced=${priced}`
