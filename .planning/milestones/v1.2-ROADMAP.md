# Milestone v1.2: Dashboard & Governance Platform

**Status:** SHIPPED 2026-02-28
**Phases:** 10-15
**Total Plans:** 14

## Overview

Turn Govyn from a CLI-only proxy into a visual governance platform where both devs and team leads can monitor costs, manage policies, approve agent actions, and configure alerts. Separate React + TypeScript + Tailwind dashboard with Clerk auth, PostgreSQL (Neon) persistence, and full-stack approval queue.

## Phases

### Phase 10: Data Persistence & Proxy API

**Goal**: Proxy persists all governance data to PostgreSQL and exposes API endpoints for the dashboard to consume, including the approval queue backend
**Depends on**: Phase 9.1 (v1.1 complete)
**Plans**: 2 plans

Plans:
- [x] 10-01: Database schema, migrations, connection pool, async DB writer for cost records and policy evaluations
- [x] 10-02: Approval queue backend — require_approval policy type, HTTP 202 flow, polling endpoint, approval token validation

**Details:**
- PostgreSQL schema with 5 tables (cost_records, policy_evaluations, approval_requests, cost_daily_summary, govyn_migrations) and 10 indexes
- Versioned migration system that auto-applies on proxy startup
- Async DbWriter with configurable fail-open/fail-closed resilience modes
- RetentionManager with pre-delete aggregation into daily summaries
- require_approval policy type with AND-match criteria
- Full approval lifecycle: HTTP 202 -> poll -> approve/deny -> re-send with single-use token
- ApprovalTimeoutChecker auto-denies expired pending approvals

## Milestone Summary

**Key Decisions:**
- Dashboard as separate React app, not embedded in proxy (ADR-010)
- PostgreSQL via Neon for persistence (ADR-012)
- Clerk auth for dashboard
- Approval queue uses HTTP 202 + polling pattern (ADR-017)
- Alerts via webhook only (no email in v1.2)
- postgres (porsager/postgres) for ESM-native zero-dependency PostgreSQL client
- Fire-and-forget DB writes for zero latency on hot path
- Tailwind v4 via Vite plugin (no PostCSS config)
- CodeMirror 6 (not Monaco) for lightweight YAML editing
- Sonner for toast notifications

**Issues Resolved:**
- shadcn CLI component output path issue (literal @/ instead of alias resolution)
- Recharts v3 tooltip type compatibility
- npm peer dependency conflict with --legacy-peer-deps
- noUncheckedIndexedAccess array access issues

**Issues Deferred:**
- Historical time-series cost API (charts use snapshot data from aggregated response)
- CSV/JSON cost export
- Bulk approval actions
- Policy dry-run testing
- Policy version history
- Email alert delivery

**Technical Debt Incurred:**
- Chart shows snapshot data until historical time-series API is built
- Last Active column in agent table shows placeholder (API doesn't expose timestamps)
- window.confirm used for alert delete confirmation (vs custom modal elsewhere)
