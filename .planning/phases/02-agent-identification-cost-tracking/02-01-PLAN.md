4. Create src/agents.ts with:
   - `resolveAgentId(req: IncomingMessage, agents: Map<string, AgentConfig>): AgentIdentity`
   - Resolution order per ADR-014:
     a. Check `x-govyn-agent` header (case-insensitive lookup). If present, return `{ agentId: headerValue, source: 'header' }`. Do NOT validate that the agent exists in config — any string is valid (agents can self-identify).
     b. Check `authorization` header for Bearer token. Look up in agents map to see if any agent has this key in its `apiKeys` array. If matched, return `{ agentId: matchedAgentName, source: 'api-key' }`.
     c. Default: return `{ agentId: 'unknown', source: 'default' }`.

5. Create tests/agents.test.ts with tests:

   - Request with X-GOVYN-AGENT (uppercase) resolves correctly (headers are case-insensitive)
   - Request with API key matching an agent's apiKeys resolves to that agent
   - Request with API key not matching any agent resolves to 'unknown'
   - Request with no header and no matching key resolves to 'unknown' with source 'default'
   - Request with both header and matching API key prefers header (header takes priority)
   - Empty agents map always resolves header-based or defaults to 'unknown'
  </action>

<task type="auto">
  <name>Task 2: Token extraction from OpenAI and Anthropic response bodies</name>
  <files>src/tokens.ts, src/types.ts, tests/tokens.test.ts</files>
  <action>
1. Add to src/types.ts:
   - `TokenUsage` interface: `{ inputTokens: number; outputTokens: number; totalTokens: number; model: string; provider: ProviderType }`

2. Create src/tokens.ts with:
   - `extractTokenUsage(responseBody: string, provider: ProviderType): TokenUsage | null`
   - For OpenAI (provider === 'openai'): Parse JSON body, read `usage.prompt_tokens` as inputTokens, `usage.completion_tokens` as outputTokens, `usage.total_tokens` as totalTokens, `model` from root. OpenAI response shape: `{ ..., model: "gpt-4o", usage: { prompt_tokens: N, completion_tokens: N, total_tokens: N } }`
   - For Anthropic (provider === 'anthropic'): Parse JSON body, read `usage.input_tokens` as inputTokens, `usage.output_tokens` as outputTokens, compute totalTokens as sum, `model` from root. Anthropic response shape: `{ ..., model: "claude-sonnet-4-20250514", usage: { input_tokens: N, output_tokens: N } }`
   - For custom providers: attempt OpenAI-format parsing first (most OpenAI-compatible endpoints use this shape). If no usage field found, return null.
   - If JSON parsing fails or usage fields are missing, return null (never throw).

   - `extractTokenUsageFromSSE(chunks: string[], provider: ProviderType): TokenUsage | null`
   - For OpenAI SSE: The final data chunk before `[DONE]` contains `usage` field when `stream_options: { include_usage: true }` is set. Parse the last non-[DONE] data line as JSON, extract usage. Also check for standalone usage chunks (OpenAI sends a final chunk with usage only).
   - For Anthropic SSE: Look for `event: message_delta` events that contain `usage: { output_tokens: N }`, and `event: message_start` which contains `usage: { input_tokens: N }`. Parse both and combine.
   - If no usage data found in any chunk, return null.
   - NOTE: For Phase 2 MVP, the proxy will extract tokens from non-streaming responses by buffering the response body. SSE token extraction will read the accumulated chunks. Full streaming token extraction without any buffering is a Phase 5 optimization if needed.

3. Create tests/tokens.test.ts with tests covering:
   - OpenAI non-streaming response body: correct token extraction
   - OpenAI response with missing usage field returns null
   - Anthropic non-streaming response body: correct token extraction
   - Anthropic response with missing usage field returns null
   - Custom provider with OpenAI-compatible response: extracts tokens
   - Custom provider with no usage field: returns null
   - Invalid JSON input: returns null (no throw)
   - OpenAI SSE chunks with usage in final chunk: correct extraction
   - Anthropic SSE chunks with message_start + message_delta usage: correct extraction
   - SSE chunks with no usage data: returns null
   - Use realistic example response bodies from OpenAI and Anthropic documentation formats
  </action>
  <verify>
    <automated>npx vitest run tests/tokens.test.ts --reporter=verbose 2>&1</automated>
  </verify>
  <done>Token extraction works for both OpenAI and Anthropic non-streaming and SSE responses. Returns null safely on missing/malformed data. All 10+ tests pass.</done>
</task>

<task type="auto">
  <name>Task 3: Pricing table and cost calculation engine</name>
  <files>src/pricing.ts, src/config.ts, govyn.config.yaml, tests/pricing.test.ts</files>
  <action>
1. Create src/pricing.ts with:
   - `ModelPricing` interface: `{ model: string; inputPricePerMillion: number; outputPricePerMillion: number }`
   - `PricingTable` type: `Map<string, ModelPricing>` (model name -> pricing)
   - `CostResult` interface: `{ inputCost: number; outputCost: number; totalCost: number; model: string; priced: boolean }`

   - `getDefaultPricing(): PricingTable` — returns built-in pricing for major models (all prices per 1M tokens, sourced from current public pricing as of February 2026):
     - gpt-4o: input $2.50, output $10.00
     - gpt-4o-mini: input $0.15, output $0.60
     - gpt-4.1: input $2.00, output $8.00
     - gpt-4.1-mini: input $0.40, output $1.60
     - gpt-4.1-nano: input $0.10, output $0.40
     - o3: input $2.00, output $8.00
     - o3-mini: input $1.10, output $4.40
     - o4-mini: input $1.10, output $4.40
     - claude-sonnet-4-20250514: input $3.00, output $15.00
     - claude-haiku-3-5-20241022: input $0.80, output $4.00
     - claude-opus-4-20250514: input $15.00, output $75.00
     NOTE: These are built-in defaults. Config overrides take precedence.

   - `loadPricing(configPricing?: Record<string, { input: number; output: number }>): PricingTable`
     - Start with getDefaultPricing(), overlay any config-defined pricing (config prices override defaults for same model name). Config format uses per-million-token pricing.

   - `calculateCost(usage: TokenUsage, pricingTable: PricingTable): CostResult`
     - Look up model in pricing table. If found: compute inputCost = (inputTokens / 1_000_000) * inputPricePerMillion, same for output. Set priced = true.
     - If model NOT found in table: log a warning `[govyn] WARNING: Unknown model "${model}" — cost marked as unpriced`, return totalCost = 0, priced = false.
     - COST-08 accuracy: This approach matches provider billing exactly for non-streaming (we use the same token counts the provider reports). For streaming, accuracy depends on token extraction from SSE chunks.

2. Update src/config.ts to parse optional `pricing` section from YAML config:
   ```yaml
   pricing:
     gpt-4o:
       input: 2.50   # per million tokens
       output: 10.00
     my-custom-model:
       input: 1.00
       output: 5.00
   ```
   Add pricing field to RawConfig interface. Pass raw pricing data to loadPricing(). Store PricingTable on a new exported config object or return it alongside ProxyConfig. Approach: add `pricing: PricingTable` to ProxyConfig type.

3. Update govyn.config.yaml to include a commented-out pricing section with example:


   - calculateCost with known model returns correct cost (test with specific token counts, verify math)
   - calculateCost with gpt-4o, 1000 input tokens, 500 output tokens = (1000/1M * 2.50) + (500/1M * 10.00) = 0.0025 + 0.005 = 0.0075
   - calculateCost with unknown model returns totalCost 0, priced false
   - loadPricing with config overrides: config pricing overrides default for same model
   - loadPricing with new model in config: adds to pricing table
   - loadPricing with no config: returns only defaults
   - Verify warning is logged for unknown models (capture console.warn)
