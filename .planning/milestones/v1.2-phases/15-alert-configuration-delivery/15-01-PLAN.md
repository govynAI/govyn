    **3. Create src/alert-manager.ts:**

    ```typescript
    import type postgres from 'postgres';
    import { govynEvents, type GovynEvent } from './events.js';

    export interface AlertRule {
      id: string;
      name: string;
      type: 'budget_threshold' | 'policy_trigger';
      enabled: boolean;
      config: BudgetThresholdConfig | PolicyTriggerConfig;
      webhookUrl: string;
      cooldownMinutes: number;
      lastFiredAt: Date | null;
      createdAt: Date;
      updatedAt: Date;
    }

    export interface BudgetThresholdConfig {
      agent_id: string;    // specific agent or '*' for all
      metric: 'daily' | 'monthly';
      threshold_percent: number;
    }

    export interface PolicyTriggerConfig {
      policy_name: string; // specific policy name or '*' for any
      agent_id: string;    // specific agent or '*' for all
    }

    export interface AlertHistoryEntry {
      id: string;
      ruleId: string;
      ruleName: string;
      ruleType: string;
      eventType: string;
      eventPayload: Record<string, unknown>;
      webhookUrl: string;
      webhookStatus: number | null;
      webhookError: string | null;
      firedAt: string;
    }
    ```

    Class: `AlertManager`
    - Constructor takes `sql: postgres.Sql`
    - `start()`: loads rules from DB, subscribes to `govynEvents.on('event', this.handleEvent)`
    - `stop()`: unsubscribes from events
    - `handleEvent(event: GovynEvent)`: for each enabled rule with no cooldown active:
      - budget_threshold rules: match `budget_warning` and `budget_exceeded` events where percentUsed >= threshold AND (agent_id matches or is '*') AND metric matches limitPeriod
      - policy_trigger rules: match `policy_denied` events where (policy_name matches or is '*') AND (agent_id matches or is '*')
      - On match: call `fireAlert(rule, event)`
    - `fireAlert(rule, event)`:
      1. Send HTTP POST to rule.webhookUrl with JSON body: `{ alert: { rule_id, rule_name, rule_type }, event: { type, ...eventData }, fired_at: ISO timestamp }`
      2. Record in alert_history: rule_id, rule_name, rule_type, event_type, event_payload (full event), webhook_url, webhook_status (HTTP status or null on network error), webhook_error (error message or null)
      3. Update alert_rules.last_fired_at = NOW()
      4. Emit `alert_fired` event on govynEvents
      5. Use try/catch: if webhook fails, still record history with error info
    - `reloadRules()`: re-fetch all rules from DB (called after CRUD operations)
    - Cooldown logic: skip rule if `last_fired_at` + `cooldown_minutes` > now. Use in-memory cache of lastFiredAt per rule ID to avoid DB reads in hot path. Update cache when firing.

    Webhook POST format:
    ```json
    {
      "alert": {
        "rule_id": "uuid",
        "rule_name": "High spend warning",
        "rule_type": "budget_threshold"
      },
      "event": {
        "type": "budget_warning",
        "agent_id": "research-agent",
        "percent_used": 85,
        "current_spend": 8.50,
        "limit": 10.00
      },
      "fired_at": "2026-02-28T12:00:00.000Z",
      "source": "govyn"
    }
    ```

    Use native `fetch()` for webhook delivery (Node 18+ built-in). Set a 10-second timeout via AbortController. Set User-Agent: `Govyn-Alerts/1.0`. Content-Type: `application/json`.

    **1. Create src/alert-api.ts:**

    Pattern follows src/approval-api.ts — a single exported function that handles all /api/alerts/* routes.

    ```typescript
    import type http from 'node:http';
    import type postgres from 'postgres';
    import type { AlertManager } from './alert-manager.js';

    export function handleAlertApi(
      req: http.IncomingMessage,
      res: http.ServerResponse,
      sql: postgres.Sql,
      alertManager: AlertManager,
    ): void;
    ```

    Routes dispatched inside:

    **GET /api/alerts/rules** — List all alert rules
    - Query: none (return all rules, ordered by created_at DESC)
    - Response: `{ rules: AlertRule[] }`
    - Query DB: `SELECT * FROM alert_rules ORDER BY created_at DESC`
    - Map snake_case DB columns to camelCase response fields

    **POST /api/alerts/rules** — Create a new alert rule
    - Body: `{ name: string, type: 'budget_threshold' | 'policy_trigger', config: object, webhook_url: string, cooldown_minutes?: number, enabled?: boolean }`
    - Validate: name required (non-empty), type must be valid, config shape validated per type, webhook_url required (must start with http:// or https://)
    - For budget_threshold config: require agent_id, metric (daily|monthly), threshold_percent (1-100)
    - For policy_trigger config: require policy_name, agent_id
    - INSERT into alert_rules, return created rule with 201
    - Call alertManager.reloadRules() after insert

    **PUT /api/alerts/rules/:id** — Update an existing rule
    - Body: partial update — any of { name, enabled, config, webhook_url, cooldown_minutes }
    - Build dynamic SET clause (only provided fields)
    - Set updated_at = NOW()
    - Return 404 if rule not found
    - Call alertManager.reloadRules() after update

    **DELETE /api/alerts/rules/:id** — Delete a rule
    - DELETE FROM alert_rules WHERE id = $1
    - Return 404 if no rows affected
    - Return `{ success: true }` on delete
    - Call alertManager.reloadRules() after delete

    **GET /api/alerts/history** — List alert history
    - Query params: limit (default 50, max 200), offset (default 0), rule_id (optional filter)
    - Response: `{ alerts: AlertHistoryEntry[], total: number, limit: number, offset: number }`
    - Order by fired_at DESC
    - Follow same pagination pattern as approval-api.ts using sql.unsafe for dynamic WHERE

    **POST /api/alerts/test** — Test webhook delivery
    - Body: `{ webhook_url: string }`
    - Send a test POST to the URL with a sample payload: `{ alert: { rule_name: "Test Alert" }, event: { type: "test" }, fired_at: now, source: "govyn" }`
    - Return `{ success: true, status: httpStatus }` or `{ success: false, error: message }`
    - Use 10-second timeout
