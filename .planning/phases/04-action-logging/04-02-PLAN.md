**1. Create `src/log-rotator.ts`:**

Implement `LogRotator` class:

**Constructor:** `new LogRotator(config: LoggingConfig)`. Stores config references. Sets up a cleanup interval via `setInterval` (every hour, with `.unref()` to not prevent process exit) that calls `cleanupExpired()`.

**`checkRotation(currentFilePath: string): { shouldRotate: boolean; reason: 'size' | 'time' | null }`:**
- Size check: `fs.statSync(currentFilePath).size` exceeds `config.rotationMaxSizeMb * 1024 * 1024`
- Time check: Parse the date from the filename (govyn-YYYY-MM-DD.jsonl) or check file mtime. If current time minus file creation time exceeds `config.rotationIntervalHours * 3600 * 1000`, rotate.
- Returns which trigger fired (or null if no rotation needed).

**`rotate(currentFilePath: string): string`:**
- Generate rotated filename: `govyn-{YYYY-MM-DD}-{HHmmss}.jsonl.gz`
- Read current file content, compress with `zlib.gzipSync()`, write to rotated path
- Delete (unlink) the original file
- Return the path to the new current file (a fresh `govyn-{YYYY-MM-DD}.jsonl` which ActionLogger will create on next write)
- All I/O is synchronous since this runs inside flush() which is already on a timer

**`cleanupExpired(): void`:**
- Scan `config.directory` for `.jsonl.gz` files. Parse dates from filenames. Delete files older than `config.retentionDays`.
- Scan `config.directory/payloads/` for `.json` files. Check mtime. Delete files older than `config.payloadRetentionDays`.
- Log deletions to stderr: `[govyn] Cleaned up N expired log files, M expired payload files`
- Wrap in try/catch — cleanup failures should never crash the process.

**`stop(): void`:** Clears the cleanup interval.

**2. Update `src/action-logger.ts` — integrate LogRotator:**

Import `LogRotator`. In constructor, create a `LogRotator` instance.

In `flush()` method, BEFORE writing buffered entries, call `rotator.checkRotation(currentFilePath)`. If shouldRotate is true, call `rotator.rotate(currentFilePath)` and update the internal `currentFilePath` to point to a fresh file.

Update `close()` to also call `rotator.stop()`.

Expose a read-only `logDirectory` property (getter) returning `config.directory` — needed by the log query API.

Expose a `getPayloadPath(payloadId: string): string` method returning the full path to a payload file.

**3. Write unit tests in `tests/log-rotator.test.ts`:**

Tests:

- `checkRotation()` returns shouldRotate=true with reason='size' when file exceeds max size

- `rotate()` creates a .jsonl.gz file and the original is removed
- `rotate()` output is valid gzip (decompress and verify content matches)
- `cleanupExpired()` deletes log files older than retentionDays
- `cleanupExpired()` deletes payload files older than payloadRetentionDays
- `cleanupExpired()` keeps files within retention period
- `cleanupExpired()` handles empty directories gracefully
- `stop()` clears the cleanup interval

**1. Create `src/log-api.ts`:**

Implement `handleLogApi()` function following the same pattern as `handleBudgetApi()` and `handleCostApi()`.

**Routes:**
- `GET /api/logs` — List log entries with filtering and cursor-based pagination
- `GET /api/logs/:id` — Get a single log entry by ID
- `GET /api/logs/:id/payload` — Get stored payload for a log entry

**GET /api/logs query parameters:**
- `agent` — Filter by agent_id (exact match)
- `status` — Filter by HTTP status code (exact match, e.g., `status=200` or `status=429`)
- `start` — Filter by timestamp >= start (ISO 8601 string)
- `end` — Filter by timestamp <= end (ISO 8601 string)
- `model` — Filter by model name (exact match)
- `provider` — Filter by provider type (exact match)
- `limit` — Max entries per page (default 50, max 200)
- `cursor` — Opaque cursor token for pagination (base64-encoded `{file}:{line}` offset)

**Implementation for GET /api/logs:**
- Constructor takes `logDirectory: string` (the ActionLogger's log directory)
- Read JSONL files in reverse chronological order (newest first): list `.jsonl` files, sort by name descending
- For each file, read lines and parse as JSON. Apply filters. Collect up to `limit + 1` results (extra one to determine if there's a next page).
- If cursor provided: decode to `{file}:{line}`, skip entries until reaching that position.
- Response format:
  ```json
  {
    "entries": [ ...LogEntry objects... ],
    "cursor": "base64_encoded_position_or_null",
    "has_more": boolean
  }
  ```
- If there are `limit + 1` results, return `limit` entries plus `cursor` and `has_more: true`.

**GET /api/logs/:id:**
- Search through JSONL files for an entry where `id` matches.
- Return the full LogEntry object.
- 404 if not found.

**GET /api/logs/:id/payload:**
- First find the log entry to get the `payload_id`.
- If `has_payload` is false, return 404 with message "No payload stored for this log entry".
- Read the payload file from `{logDirectory}/payloads/{payload_id}.json`.
- Return the payload JSON content.
- 404 if payload file is missing (may have been cleaned up by retention).

**Non-GET methods:** Return 405.

Helper function `sendJson(res, statusCode, body)` — same pattern as budget-api.ts.

Parse URL query parameters using `new URL(url, 'http://localhost').searchParams`.
