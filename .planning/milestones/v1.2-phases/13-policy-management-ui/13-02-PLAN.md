**1. Create `dashboard/src/hooks/usePolicy.ts`:**

A hook for single-policy operations (view, save, delete, toggle, validate).

```typescript
interface UsePolicyResult {
  policy: PolicyDetail | null;
  loading: boolean;
  error: string | null;
  saving: boolean;
  validationErrors: PolicyValidationError[];
  refetch: () => void;
  save: (yaml: string) => Promise<boolean>;
  deletePolicy: () => Promise<boolean>;
  toggleEnabled: (enabled: boolean) => Promise<boolean>;
  validateYaml: (yaml: string) => void;
}
```

Implementation:
- `fetchPolicy(name)`: calls `apiFetch('/api/policies/${name}')`, returns `PolicyDetail`.
- `save(yaml)`: calls `PUT /api/policies/${name}` with `{ yaml }`. On 400, parse response body for `errors` array and set `validationErrors`. On success, refetch and return true.
- `deletePolicy()`: calls `DELETE /api/policies/${name}`. On success, return true. Caller navigates to `/policies`.
- `toggleEnabled(enabled)`: calls `PATCH /api/policies/${name}` with `{ enabled }`. Refetch after.
- `validateYaml(yaml)`: calls `PUT /api/policies/${name}` endpoint but with a query param `?validate=true` — OR, better: do client-side validation. Since PolicyParser runs on the proxy (Node.js, uses `fs`), we can't run it in the browser. Instead, use a dedicated validation endpoint: call `POST /api/policies/validate` (to add in this plan — see below). Actually, to keep it simple: POST the yaml to the save endpoint which already validates and returns errors. But that would save it. Instead, implement client-side YAML syntax checking only (use `yaml` package's `parseDocument` which is available in browser), and rely on server-side validation on save attempt.

  **Revised approach for validation:** Use the browser-compatible `yaml` package (install in dashboard if not present: `npm install yaml`). Parse with `parseDocument(yamlString)` in the browser to detect YAML syntax errors. Then also validate required fields client-side (check for `version: 1`, `policies` array, each policy has `name`, `type`, etc.) — a lightweight client-side validator. Server-side validation via PolicyParser remains the safety net on save.

  Create a `validatePolicyYaml(yaml: string): PolicyValidationError[]` function within the hook file or a separate utility. It should:
  1. Parse with `yaml` package's `parseDocument` — catch YAML syntax errors.
  2. Check `version` field exists and equals 1.
  3. Check `policies` array exists and has exactly 1 entry (for single-policy editing).
  4. Check each policy has `name`, `type`, `scope` (or defaults to global).
  5. Check `type` is one of the 7 valid types.
  6. Return errors with line numbers extracted from yaml parse positions.

- Debounce validation: use a 500ms debounce timer (per CONTEXT.md decision). When `validateYaml` is called, clear the previous timer, set a new one. When timer fires, run `validatePolicyYaml` and set `validationErrors`.
