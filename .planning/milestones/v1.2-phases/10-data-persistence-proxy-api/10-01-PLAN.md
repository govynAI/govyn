Add PostgreSQL persistence to the Govyn proxy so that cost records and policy evaluation results are written to a database alongside the existing in-memory and JSONL logging.

Purpose: Enable the dashboard (Phase 11+) to query historical cost data with time-windowed aggregation, and provide the foundation for the approval queue (Plan 02). The proxy must remain operational when the database is unavailable (fail-open by default, configurable to fail-closed).

Output: Database connection module, schema with migrations, async DB writer, retention manager, and integration with the existing proxy request flow.
</objective>

    **Create `src/db-writer.ts`** — DbWriter class:

    ```typescript
    export class DbWriter {
      constructor(private sql: postgres.Sql, private failOpen: boolean) {}

      // Write cost record — fire-and-forget async, never blocks hot path
      async writeCostRecord(record: CostRecord): Promise<void> {
        try {
          await this.sql`
            INSERT INTO cost_records (agent_id, model, provider, input_tokens, output_tokens, input_cost, output_cost, total_cost, priced, requested_model, created_at)
            VALUES (${record.agentId}, ${record.model}, ${record.provider}, ${record.inputTokens}, ${record.outputTokens}, ${record.inputCost}, ${record.outputCost}, ${record.totalCost}, ${record.priced}, ${record.requestedModel ?? null}, ${new Date(record.timestamp)})
          `;
        } catch (err) {
          if (this.failOpen) {
            process.stderr.write(`[govyn] DB write failed (fail-open): ${err instanceof Error ? err.message : String(err)}\n`);
          } else {
            throw err; // fail-closed: propagate
          }
        }
      }

      // Write policy evaluation result
      async writePolicyEvaluation(eval: { agentId, provider, path, allowed, evaluatedCount, matchedCount, deniedBy?, deniedReason?, evaluationTimeMs }): Promise<void> {
        // Same try/catch pattern with fail-open
      }

      // Check if DB is available (for approval flow requirement: approvals ALWAYS need DB)
      async isAvailable(): Promise<boolean> {
        try {
          await this.sql`SELECT 1`;
          return true;
        } catch { return false; }
      }
    }
    ```

    Key design decisions:
    - All write methods are async but called with `.catch()` (fire-and-forget) from the proxy hot path — zero added latency, matching the existing ActionLogger pattern
    - When `failOpen` is true, catch errors and log to stderr (matching existing JSONL log behavior during failures)
    - When `failOpen` is false, rethrow errors so the proxy can reject the request
    - The `isAvailable()` method is used by the approval flow (Plan 02) since approvals ALWAYS require DB per CONTEXT.md
