**API endpoint to implement:**

1. **GET /api/approvals** — List approval requests with filtering and pagination.

   Query parameters:
   - `status` — Filter by status: `pending`, `approved`, `denied`, `denied_timeout`, or `all` (default: `all`). Support comma-separated for multiple: `pending,approved`.
   - `limit` — Number of results per page (default: 50, max: 200).
   - `offset` — Pagination offset (default: 0).
   - `agent_id` — Optional filter by agent ID (exact match).

   Response shape:
   ```json
   {
     "approvals": [
       {
         "id": "uuid",
         "agent_id": "agent-name",
         "provider": "openai",
         "model": "gpt-4",
         "target_path": "/v1/chat/completions",
         "policy_name": "require-approval-expensive",
         "estimated_cost": 0.05,
         "request_summary": "User asked about...",
         "status": "pending",
         "decided_by": null,
         "decision_notes": null,
         "decided_at": null,
         "expires_at": "2026-02-28T18:00:00.000Z",
         "created_at": "2026-02-28T17:30:00.000Z"
       }
     ],
     "total": 42,
     "limit": 50,
     "offset": 0
   }
   ```

   SQL query:
   - SELECT from `approval_requests` with optional WHERE clauses for status and agent_id.
   - ORDER BY: `pending` first (ordered by `created_at ASC` — oldest first so they're most urgent), then non-pending ordered by `decided_at DESC` (most recent decisions first). Use `CASE WHEN status = 'pending' THEN 0 ELSE 1 END, CASE WHEN status = 'pending' THEN created_at END ASC, decided_at DESC NULLS LAST` for ordering.
   - Count total matching rows with a separate `SELECT COUNT(*)` query (or use a CTE with window function).
   - Apply LIMIT and OFFSET for pagination.
   - Do NOT return `request_payload`, `approval_token`, or `token_used` in the list response (these are large/sensitive fields).
   - Cast `estimated_cost` from NUMERIC to number via `parseFloat`.
   - Format dates as ISO strings.

   Implementation:
   ```typescript
   import type http from 'node:http';
   import type postgres from 'postgres';

   export function handleApprovalApi(
     req: http.IncomingMessage,
     res: http.ServerResponse,
     sql: postgres.Sql,
   ): void {
     // Parse query params from req.url
     // Build parameterized SQL query
     // Return JSON response
   }
   ```

   Parse URL query params using `new URL(req.url!, 'http://localhost')`. Validate `limit` is between 1 and 200. Validate `offset` is >= 0. For `status`, split on commas and validate each value is one of the valid statuses.

   Error handling:
   - Return 400 for invalid query parameters.
   - Return 500 for database errors (catch and log).

- Return `{ approvals, total, loading, error, refetch }`.
