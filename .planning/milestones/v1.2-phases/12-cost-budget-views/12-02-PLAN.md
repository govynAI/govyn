This component needs to handle TWO modes:
- **Stacked mode** (overview page): Multiple agents as stacked areas, each a distinct color
- **Single mode** (agent detail page): One agent's cost as a single filled area

Props:
```typescript
interface CostChartDataPoint {
  label: string;        // x-axis label (e.g., "Feb 27", "Week 8")
  total: number;        // total cost at this point
  [agentId: string]: number | string;  // per-agent cost values for stacking
}

interface CostAreaChartProps {
  data: CostChartDataPoint[];
  agents?: string[];      // agent IDs for stacked mode (determines areas + colors)
  height?: number;        // chart height in px (default: 350)
  stacked?: boolean;      // true for overview stacked chart, false for single agent
}
```

Implementation:
- Wrap in `<ResponsiveContainer width="100%" height={height}>` for responsive sizing
- Use `<AreaChart data={data}>` with `<CartesianGrid strokeDasharray="3 3" />` for grid lines
- XAxis: `dataKey="label"`, tick styling with muted foreground color, `fontSize={12}`
- YAxis: format values as dollar amounts `$X.XX`, tick styling same as XAxis
- For stacked mode: render one `<Area>` per agent with:
  - `type="monotone"` for smooth curves
  - `stackId="costs"` for stacking
  - `fill` and `stroke` from a color palette (use 6-8 distinct colors that work in both dark/light themes — teal, blue, purple, orange, pink, cyan, lime, rose)
  - `fillOpacity={0.4}` for translucent fill
- For single mode: one `<Area>` with teal fill (matching the primary accent color `#14b8a6`)
- Custom `<Tooltip>` with interactive hover per user decision:
  - Content wrapper: bg-[var(--popover)] text-[var(--popover-foreground)] border rounded-lg shadow-lg p-3
  - Show date, total cost, and per-agent breakdown when stacked
  - Format costs as $X.XXXX
- Style grid lines to use `var(--border)` color for theme compatibility

**IMPORTANT: Data generation helper.** The proxy API returns AGGREGATED totals per period, NOT time-series data points. The API does not return historical daily/weekly buckets — it returns a single summary for the selected period. For MVP, the chart will show a SINGLE data point (the current period total) which isn't very useful for a time-series. To handle this gracefully:

- Accept that the chart will be sparse until historical data accumulates
- Generate the chart data from the `agents` array in the API response — each agent becomes one data point showing their contribution
- For the "stacked overview" chart, create a simple bar-like display showing per-agent contribution rather than a true time-series (since we only have one time point)
- Add a comment in the code: `// TODO: When proxy API supports historical time-series data (e.g., /api/costs/timeseries), replace this with real temporal data`
- For now, if there's only one data point, render it as a minimal chart that at least shows the data exists

Alternative approach (better UX): Since we have the `generated_at` timestamp and the data, create a "snapshot" chart that shows agent spend as stacked bars at the current time point. The chart component should still be built to support multiple time points — it will render correctly once the API provides them.
