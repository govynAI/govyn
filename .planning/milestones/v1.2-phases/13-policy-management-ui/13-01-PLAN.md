  <action>
Create `src/policy-api.ts` with a `handlePolicyApi` function that handles all policy CRUD operations. The function receives `req`, `res`, `policyEngine`, and `configPoliciesFile` (the path to the YAML file from config).

**API endpoints to implement:**

1. **GET /api/policies** â€” List all policies. Response: `{ policies: PolicySummary[] }` where PolicySummary is `{ name, type, scope: { level, value? }, enabled, description? }`. Read from `policyEngine.getPolicies()`.

2. **GET /api/policies/:name** â€” Get single policy detail. Response: the full policy object from `policyEngine.getPolicies().find(p => p.name === name)` plus a `yaml` field containing the single policy serialized as YAML (use the `yaml` package's `stringify` to convert the policy object to YAML). Return 404 if not found.

3. **PATCH /api/policies/:name** â€” Toggle enabled/disabled. Body: `{ enabled: boolean }`. Read the policy YAML file from disk, parse with `yaml` package's `parseDocument`, find the policy node by name in the `policies` array, update its `enabled` field, write back to disk, then reload via `policyEngine.loadFromFile()`. Return the updated policy summary. Return 404 if policy not found.

4. **PUT /api/policies/:name** â€” Update policy YAML. Body: `{ yaml: string }`. Parse the submitted YAML with `parsePolicies()` (from policy-parser.ts) for validation. If validation fails, return 400 with `{ errors: PolicyParseError[] }`. If valid, read the full policy file from disk, find and replace the policy entry by name, write back, reload engine. Return the updated policy. Return 404 if policy not found.

5. **POST /api/policies** â€” Create new policy. Body: `{ yaml: string }`. Validate with `parsePolicies()`. If valid, read the full policy file from disk, append the new policy entry to the `policies` array, write back, reload engine. Return 201 with the new policy. Return 400 if validation fails or name conflicts.

6. **DELETE /api/policies/:name** â€” Remove a policy. Read the full policy file, remove the matching policy entry, write back, reload engine. Return `{ success: true }`. Return 404 if not found.

**Important implementation details:**
- Use `import { parseDocument, stringify } from 'yaml'` (already in package.json dependencies for the proxy).
- For PATCH/PUT/POST/DELETE: read the YAML file from `configPoliciesFile` path, modify the parsed document, write it back with `fs.writeFileSync`. The PolicyWatcher will auto-detect the file change and trigger hot reload, but also call `policyEngine.loadFromFile()` explicitly for immediate consistency.
- The `yaml` field in GET /api/policies/:name should wrap the single policy in a `version: 1\npolicies:\n  - ...` structure so the dashboard editor can validate it as a complete policy document.
- URL-decode the policy name from the URL path for names with special characters.
- For POST/PUT, extract the single policy from the submitted yaml (which should be a valid `version: 1, policies: [...]` document with exactly one policy).

**Wire into server.ts:**
- Add `import { handlePolicyApi } from './policy-api.js';` at the top.
- Add route matching before the proxy routing section: `if (url.startsWith('/api/policies') && policyEngine) { handlePolicyApi(req, res, policyEngine, config.policiesFile ?? ''); return; }`
- Update CORS `Access-Control-Allow-Methods` to include `PUT, PATCH, DELETE` (add to the existing list in the CORS section).

**Wire into index.ts:**
- No changes needed. The `policyEngine` and `config.policiesFile` are already passed to `startServer`. However, the `startServer` function needs `policiesFile` passed through. Update the `startServer` function signature in server.ts to accept `policiesFile?: string` as a parameter, and pass `config.policiesFile` from index.ts.
