  <action>
    1. **Install shadcn/ui components:**
       `npx shadcn@latest add popover badge input label card`

    2. **Create `api-client.ts`:**
       - Class or module with a configurable base URL
       - `getBaseUrl()`: reads from localStorage key `'govyn-proxy-url'`, returns string or null
       - `setBaseUrl(url: string)`: writes to localStorage
       - `ping()`: `GET {baseUrl}/health` ظ¤ returns `{ ok: boolean, latencyMs: number, data?: { status, version, uptime_seconds } }`. On fetch error or timeout (5s), returns `{ ok: false, latencyMs: -1 }`.
       - `fetch(path: string, options?)`: wrapper around native fetch that prepends baseUrl and handles common error patterns. For future use by Phases 12-15.
       - All methods handle missing baseUrl gracefully (return disconnected state, not throw)

    3. **Create `ProxyConnectionContext.tsx`:**
       - React context providing connection state to entire app
       - State: `{ status: 'connected' | 'disconnected' | 'reconnecting', proxyUrl: string | null, latencyMs: number, lastPingAt: Date | null, version: string | null }`
       - On mount: read proxy URL from localStorage, start health ping interval
       - Ping interval: every 15 seconds when connected, every 5 seconds when disconnected (faster retry)
       - State transitions:
         - `connected` ظْ ping fails ظْ `reconnecting` (first failure)
         - `reconnecting` ظْ 3 consecutive failures ظْ `disconnected`
         - `disconnected` or `reconnecting` ظْ ping succeeds ظْ `connected`
       - Expose `reconnect()` method to manually trigger an immediate ping
       - Expose `setProxyUrl(url)` to update URL and trigger immediate connection attempt

    4. **Create `useProxyConnection.ts` hook:**
       - Consumes ProxyConnectionContext
       - Returns `{ status, proxyUrl, latencyMs, lastPingAt, version, reconnect, setProxyUrl, isConnected, isDisconnected }`

    5. **Create `ConnectionStatus.tsx`:**
       Per user decision: "Status indicator in sidebar footer ظ¤ small colored dot + text, always visible"
       - Three states:
         - Connected: green dot (#22c55e) + "Connected" text
         - Disconnected: red dot (#ef4444) + "Disconnected" text
         - Reconnecting: yellow dot (#eab308) with CSS pulse animation + "Reconnecting..." text
       - When sidebar collapsed: only colored dot visible (no text)
       - When no proxy URL configured: show "Not configured" with gray dot
       - Entire status indicator is clickable ظْ opens ConnectionPopover

    6. **Create `ConnectionPopover.tsx`:**
       Per user decision: "Clicking status dot opens popover with: proxy URL, latency, last successful ping, and a Reconnect button"
       - Uses shadcn Popover component
       - Content:
         - **Proxy URL:** displayed (or "Not configured")
         - **Status:** Connected/Disconnected/Reconnecting with colored badge
         - **Latency:** "{N}ms" (or "--" if disconnected)
         - **Last ping:** relative time ("2s ago", "15s ago") or "Never"
         - **Version:** proxy version from /health response (or "--")
         - **Reconnect button:** calls `reconnect()`, disabled when already reconnecting
       - Compact, dense layout matching dev-tool aesthetic

    7. **Update `Sidebar.tsx`:**
       - Add ConnectionStatus component at the bottom of the sidebar, between the user menu and the nav items
       - Positioned just above the user section

    8. **Create `SettingsPage.tsx` (full implementation):**
       Per user decision: "Proxy API URL configured in a Settings page (stored per-user), not hardcoded via env var"
       - Replace the stub with a real settings page
       - **Proxy Connection section:**
         - Input field for proxy URL (e.g., `http://localhost:4000`)
         - "Test Connection" button that pings the URL and shows result
         - "Save" button that stores URL via `setProxyUrl()` from context
         - Show current connection status inline
       - Use shadcn Card, Input, Label, Button components
       - PageHeader with title "Settings"

    9. **Wire disconnected banner:**
       - In AppLayout, when status is `disconnected`, show a prominent banner at the top of the main content area:
         "Proxy disconnected ظ¤ data may be stale"
       - Banner should be dismissible but reappears on next navigation
       - Use yellow/amber warning styling

    10. **Update `main.tsx`:**
        - Wrap app in `<ProxyConnectionProvider>`
        - Component tree: `ThemeProvider > ClerkProvider > BrowserRouter > ProxyConnectionProvider > App`

    IMPORTANT: The proxy (port 4000) and dashboard (port 5173) are different origins. The api-client must handle CORS. For dev, Vite proxy config in `vite.config.ts` can forward `/api` requests to localhost:4000. For production, CORS headers on the proxy or a reverse proxy will be needed. For THIS plan: configure Vite dev server proxy as a convenience, but the api-client should work with absolute URLs too (since the user configures the full URL in Settings).
