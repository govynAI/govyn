<task type="auto">
  <name>Task 1: Create PolicyWatcher with file watching, debounce, and atomic reload</name>
  <files>src/policy-watcher.ts, tests/policy-watcher.test.ts</files>
  <action>
Create `src/policy-watcher.ts` exporting a `PolicyWatcher` class:

**Constructor:** `new PolicyWatcher(engine: PolicyEngine, filePath: string, options?: { debounceMs?: number })`
- `engine` — the PolicyEngine instance to reload policies into
- `filePath` — absolute or relative path to the policy YAML file
- `debounceMs` — debounce interval (default: 200ms) to coalesce rapid file changes

**Methods:**
- `start(): void` — Begin watching the file using `fs.watch()` (Node.js built-in, no chokidar dependency needed per user decision). On change event:
  1. Debounce: if a reload is pending within `debounceMs`, reset the timer
  2. Call `engine.loadFromFile(filePath)` to parse and validate the new file
  3. If `result.success === true`: log `[govyn] Policy file reloaded: N policies loaded from {path}` and emit a `policy_reloaded` event via govynEvents
  4. If `result.success === false`: log `[govyn] Policy reload failed (keeping previous policies): {first error message}` and emit a `policy_reload_failed` event via govynEvents. Do NOT call `engine.loadFromPolicies()` — the engine's `loadFromFile()` already only replaces policies on success.
- `stop(): void` — Close the fs.watch handle and clear any pending debounce timer. Idempotent (safe to call multiple times).
- `isWatching(): boolean` — Returns true if the watcher is active.

**Events to add in src/events.ts (add to GovynEvent union):**
- `{ type: 'policy_reloaded'; filePath: string; policyCount: number }`
- `{ type: 'policy_reload_failed'; filePath: string; error: string }`

**Unit tests in `tests/policy-watcher.test.ts`:**
1. Watcher detects file change and reloads policies (write temp file, modify, assert engine has new policies)
2. Invalid YAML change keeps previous policies (load valid, write invalid, assert engine still has original)
3. Debounce coalesces rapid changes (trigger 3 rapid changes, assert only 1 reload happens)
4. stop() cleans up watcher (call stop, modify file, assert no reload)
5. start() on nonexistent file logs warning but does not crash

Use `fs.mkdtempSync` + `fs.writeFileSync` for temp policy files in tests. Use `vi.useFakeTimers()` for debounce testing. Clean up temp files in afterEach.

**Important:** Use `fs.watch()` not `fs.watchFile()` — fs.watch is event-driven and faster (sub-second detection). Handle the known fs.watch quirk on some OSes where 'rename' events fire on save by treating both 'change' and 'rename' events as triggers for reload.
