  <name>Task 2: Dark/light theme system with CSS variables and localStorage persistence</name>
  <files>
    dashboard/src/index.css

  <action>
    1. **Create theme CSS variables in `index.css`:**
       Define a comprehensive set of CSS custom properties for the shadcn/ui theming system. shadcn/ui expects CSS variables on `:root` and `.dark` class (or `[data-theme="dark"]`).

       Color palette per user decision ("developer-tool minimal, green/teal accent"):
       - **Dark theme (default):** Near-black backgrounds (#0a0a0a, #111111, #1a1a1a), gray text hierarchy (#fafafa, #a1a1aa, #71717a), green/teal accent (#14b8a6 for primary, or similar teal-500)
       - **Light theme:** White/near-white backgrounds (#ffffff, #fafafa, #f4f4f5), dark text (#09090b, #3f3f46, #71717a), same green/teal accent
       - Map to shadcn/ui variable names: `--background`, `--foreground`, `--card`, `--popover`, `--primary`, `--secondary`, `--muted`, `--accent`, `--destructive`, `--border`, `--input`, `--ring`, `--sidebar-*` etc.
       - Use HSL format as shadcn/ui expects (e.g., `--primary: 174 60% 40%` for teal)

    2. **Create `ThemeContext.tsx`:**
       - React context providing `{ theme, setTheme, toggleTheme }`
       - `theme` is `'dark' | 'light'`
       - Default: `'dark'` (per user decision: "Dark mode as default theme")
       - On mount: read from `localStorage.getItem('govyn-theme')`, fall back to `'dark'`
       - On change: write to `localStorage.setItem('govyn-theme', theme)` and toggle `dark` class on `document.documentElement`
       - Use the standard pattern: add/remove `'dark'` class on `<html>` element

    3. **Create `useTheme.ts` hook:**
       - Convenience hook that consumes ThemeContext
       - Returns `{ theme, setTheme, toggleTheme, isDark }`
       - `isDark` is a computed boolean for easy conditional rendering

    4. **Update `main.tsx`:**
       - Wrap app in `<ThemeProvider>` (from ThemeContext)
       - Component tree: `ThemeProvider > ClerkProvider > BrowserRouter > App`
       - ThemeProvider is outermost so theme applies to Clerk components too

    5. **Wire theme toggle in UserMenu:**
       - Import `useTheme` in UserMenu
       - Theme dropdown item shows current state (e.g., "Dark mode" with a check, or "Switch to Light")
       - Clicking toggles between dark and light

    6. **Prevent flash of wrong theme (FOUC):**
       - Add inline `<script>` in `index.html` `<head>` that reads localStorage and sets the `dark` class BEFORE React hydrates:
       ```html
       <script>
         (function() {
           var theme = localStorage.getItem('govyn-theme') || 'dark';
           if (theme === 'dark') document.documentElement.classList.add('dark');
         })();
       </script>
       ```

    NOTE on Tailwind v4: Tailwind v4 uses `@theme` directive for custom properties and `@variant dark` for dark mode. The CSS variable approach differs from v3. Ensure the dark mode configuration works with Tailwind v4's dark mode detection (typically `prefers-color-scheme` or `.dark` class selector). If Tailwind v4's approach differs, adapt accordingly — the key requirement is: `.dark` class on `<html>` toggles dark mode, CSS variables change per theme, localStorage persists the choice.
