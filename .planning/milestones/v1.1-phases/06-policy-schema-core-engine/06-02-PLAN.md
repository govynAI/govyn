```typescript
/** Context about the current request, passed to the engine for evaluation */
export interface PolicyRequestContext {
  /** The agent making the request */
  agentId: string;
  /** The target provider (e.g., 'openai', 'anthropic') */
  provider: string;
  /** The upstream path being requested */
  path: string;
  /** HTTP method */
  method: string;
  /** Request body as string (for content matching in Phase 7) */
  body?: string;
  /** Request headers (for header matching in Phase 7) */
  headers?: Record<string, string>;
}

/** Result of evaluating a single policy against a request */
export interface SinglePolicyResult {
  /** The policy that was evaluated */
  policyName: string;
  /** The policy type */
  policyType: PolicyType;
  /** Whether this policy allows the request */
  allowed: boolean;
  /** Reason for denial (if blocked) */
  reason?: string;
  /** The deny message from the policy (if any) */
  message?: string;
  /** For rate_limit type — seconds until retry is allowed */
  retryAfterSeconds?: number;
}

/** Aggregate result of evaluating all matching policies */
export interface PolicyEvaluationResult {
  /** Whether the request is allowed (true only if ALL matching policies allow) */
  allowed: boolean;
  /** Number of policies that were evaluated */
  evaluatedCount: number;
  /** Number of policies that matched the request scope */
  matchedCount: number;
  /** If denied, the first denying policy result (most-restrictive-wins) */
  denied?: SinglePolicyResult;
  /** All individual policy results */
  results: SinglePolicyResult[];
  /** Time taken for evaluation in milliseconds */
  evaluationTimeMs: number;
}
```

- **Multiple policies all allow**: Three policies that all allow — result is `allowed: true`
- **Evaluation result includes timing**: `evaluationTimeMs` is a number >= 0
- **Performance benchmark: 100 policies < 5ms**: Load 100 block policies, evaluate one request, assert evaluationTimeMs < 5
- **getPolicies returns loaded policies**: After loading, `getPolicies()` returns all policies
- **getPoliciesByType filters by type**: `getPoliciesByType('block')` returns only block policies

**Phase 6 evaluation semantics (IMPORTANT):**
In Phase 6, the engine does NOT evaluate type-specific match criteria (regex patterns, rate windows, etc. — that's Phase 7). The Phase 6 engine evaluates ONLY:
- Scope matching (global/agent/target)
- Enabled/disabled status
- For block type: if scope matches, the policy DENIES (Phase 7 adds granular match criteria)
- For all other types: if scope matches, the policy ALLOWS in Phase 6 (Phase 7 plugs in real evaluators)

This means tests for block policies should show denial on scope match, and tests for other types should show allow (skeleton behavior).

**GREEN phase: Implement the PolicyEngine class.**

Create `src/policy-engine.ts` with class `PolicyEngine`:

**Constructor:** `new PolicyEngine()` — creates engine with empty policy store.

**Methods:**

`loadFromYaml(yamlString: string): PolicyParseResult` — Parse YAML via `parsePolicies()`, store successfully parsed policies in memory. Returns the parse result so callers can check for errors.

`loadFromPolicies(policies: Policy[]): void` — Accept pre-parsed policies directly (useful for testing and programmatic policy creation).

`clearPolicies(): void` — Remove all loaded policies.

`getPolicies(): Policy[]` — Return all loaded policies.

`getPoliciesByType(type: PolicyType): Policy[]` — Filter by type.

`evaluate(context: PolicyRequestContext): PolicyEvaluationResult` — Core evaluation method:

1. Start a high-resolution timer (`performance.now()`)
2. Filter policies to those that are `enabled: true`
3. For each enabled policy, check if scope matches the request context:
   - `level: 'global'` → always matches
   - `level: 'agent'` → matches if `scope.value === context.agentId`
   - `level: 'target'` → matches if `scope.value === context.provider`
4. For each matching policy, produce a `SinglePolicyResult`:
   - **Phase 6 skeleton behavior**: `block` type → `{allowed: false, reason: 'Blocked by policy', message: policy.message}` (Phase 7 adds granular match criteria). All other types → `{allowed: true}` (Phase 7 plugs in evaluators).
5. Apply most-restrictive-wins: if ANY matching policy has `allowed: false`, overall result is `denied`.
6. Build `PolicyEvaluationResult` with:
   - `allowed`: true only if all matching policies allow
   - `evaluatedCount`: total enabled policies checked
   - `matchedCount`: policies whose scope matched
   - `denied`: first SinglePolicyResult with `allowed: false` (if any)
   - `results`: all SinglePolicyResult entries
   - `evaluationTimeMs`: elapsed time from timer

**Performance considerations (ADR-006, ADR-013):**
- All data is in-memory, zero I/O
- Use simple array iteration (V8 optimizes this well)
- Avoid object creation in hot loop — reuse where possible
- Target: 100 policies in <5ms (the benchmark test enforces this)

**Pattern to follow:**
Follow the BudgetEnforcer pattern: constructor takes config, check method returns a result object with an `allowed` boolean and metadata.
