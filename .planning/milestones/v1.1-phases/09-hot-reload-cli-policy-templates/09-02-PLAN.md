<task type="auto">
  <name>Task 1: Add `govyn policy validate` CLI subcommand with error reporting</name>
  <files>src/cli.ts</files>
  <action>
Extend the CLI dispatch in `src/cli.ts` to support `govyn policy validate <file>`:

**Command parsing:**
Add a new branch in the main dispatch (after the `init` branch):
```
if (command === 'policy') {
  const subcommand = args[1];
  if (subcommand === 'validate') {
    // handle validation
  }
}
```

**Validation handler (`handlePolicyValidate`):**
1. Extract the file path from `args[2]`. If missing, print `Usage: govyn policy validate <file>` and exit(1).
2. Check if file exists using `fs.existsSync()`. If not, print `Error: File not found: {path}` and exit(1).
3. Import and call `parsePoliciesFromFile(filePath)` from `./policy-parser.js`.
4. If `result.success === true`:
   - Print `Valid: {result.policies.length} policies found in {filePath}`
   - For each policy, print a summary line: `  - {policy.name} ({policy.type}, scope: {policy.scope.level})`
   - If `result.warnings.length > 0`, print warnings with line numbers: `Warning: {message} (line {line})`
   - Exit with code 0
5. If `result.success === false`:
   - Print `Invalid: {result.errors.length} error(s) in {filePath}`
   - For each error: print `  Error: {message}` with ` (line {line})` if line is available, and ` (column {column})` if column is available
   - Exit with code 1

**Update help text:**
Update `printHelp()` to include:
```
Commands:
  start              Start the proxy server (default)
  init               Interactive setup wizard
  policy validate    Validate a policy YAML file
```

**Important:** Use dynamic import for policy-parser to keep CLI startup fast when not validating. The existing parsePoliciesFromFile already does the heavy lifting (reads file, parses YAML, validates schema, reports line numbers). This task just wraps it in CLI UX.

<task type="auto">
  <name>Task 2: Add CLI validation tests</name>
  <files>tests/cli-validate.test.ts</files>
  <action>
Create `tests/cli-validate.test.ts` testing the `govyn policy validate` command:

**Test approach:** Use `child_process.execSync` or `child_process.exec` to spawn `npx tsx src/cli.ts policy validate <file>` and check stdout/stderr and exit codes. Use temp files created with `fs.mkdtempSync`.

**Tests:**
1. **Valid policy file:** Write a temp YAML with a valid block policy. Run validate. Assert exit code 0, stdout contains "Valid:", stdout contains policy name and type summary.
2. **Invalid YAML syntax:** Write temp file with broken YAML (unclosed quote). Run validate. Assert exit code 1, stdout/stderr contains "Invalid:" or "Error:".
3. **Schema error with line number:** Write temp YAML with a policy missing `type` field. Run validate. Assert exit code 1, output contains line number reference.
4. **Missing file:** Run validate with nonexistent path. Assert exit code 1, output contains "File not found".
5. **No file argument:** Run `govyn policy validate` with no file arg. Assert exit code 1, output contains "Usage:".
6. **Multiple policies with warning:** Write temp file with valid policies but deprecated/warning-worthy content. Assert exit code 0 (warnings don't fail), output shows policy summaries.
7. **Type-specific validation:** Write temp file with a content_filter policy that has an invalid regex pattern (e.g., `[unclosed`). Run validate. Assert the parser catches it (exit code depends on whether parser currently validates regex — if not, this test documents current behavior).

**Cleanup:** Remove temp files in afterEach/afterAll.

**Important:** Tests must handle Windows paths correctly. Use `path.join()` for all file paths. Use `{ encoding: 'utf8' }` in execSync. Wrap execSync in try/catch to capture exit codes (execSync throws on non-zero exit).
