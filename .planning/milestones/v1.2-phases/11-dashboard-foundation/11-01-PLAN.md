  <name>Task 1: Create dashboard app with Vite + React + TypeScript + Tailwind + shadcn/ui</name>
  <files>
    dashboard/package.json

    dashboard/src/index.css

    dashboard/.env.example
    dashboard/.gitignore
  </files>
  <action>
    Create `dashboard/` directory at the repo root as a standalone app (separate from the proxy `package.json`).

    1. **Initialize Vite React-TS app:**
       - Run `npm create vite@latest dashboard -- --template react-ts` from the repo root (or manually create the equivalent files)
       - Ensure `package.json` has: react, react-dom, @types/react, @types/react-dom, vite, typescript, @vitejs/plugin-react

    2. **Install and configure Tailwind CSS v4:**
       - Install: `tailwindcss @tailwindcss/vite` (Tailwind v4 uses Vite plugin, not PostCSS)
       - Add `@tailwindcss/vite` plugin to `vite.config.ts`
       - In `src/index.css`: use `@import "tailwindcss"` (Tailwind v4 syntax)
       - No `tailwind.config.ts` or `postcss.config.js` needed with Tailwind v4 Vite plugin

    3. **Install and configure shadcn/ui:**
       - Install: `npx shadcn@latest init` (follow prompts or configure manually)
       - Create `components.json` with: style "new-york", tailwindCSS config, aliases for `@/components`, `@/lib`, `@/hooks`
       - Create `src/lib/utils.ts` with the standard `cn()` helper using `clsx` + `tailwind-merge`
       - Install utility deps: `clsx`, `tailwind-merge`, `class-variance-authority`
       - Configure path aliases in `tsconfig.app.json`: `"@/*": ["./src/*"]`
       - Configure path alias resolution in `vite.config.ts` using `resolve.alias`

    4. **Install Clerk:**
       - Install: `@clerk/clerk-react`
       - Wrap app in `ClerkProvider` in `src/main.tsx` with `publishableKey` from `import.meta.env.VITE_CLERK_PUBLISHABLE_KEY`

    5. **Install React Router:**
       - Install: `react-router-dom` (v6+)

    6. **Create `.env.example`:**
       ```
       VITE_CLERK_PUBLISHABLE_KEY=pk_test_your_key_here
       ```

    7. **Create `.gitignore`:**
       Standard Vite .gitignore + `node_modules/`, `dist/`, `.env`, `.env.local`

    8. **Verify:** `cd dashboard && npm install && npm run dev` starts without errors. The entry `src/main.tsx` renders a minimal React app wrapped in ClerkProvider.

    IMPORTANT: This is a SEPARATE app from the proxy. It has its own package.json, its own node_modules, its own dev server. The proxy runs on port 4000; the dashboard dev server runs on port 5173 (Vite default).
  </action>
  <verify>
    <automated>cd dashboard && npm install && npx tsc --noEmit && npm run build</automated>
    <manual>Run `npm run dev` in dashboard/ — browser opens to localhost:5173 without errors</manual>
  </verify>
  <done>
    dashboard/ directory exists with working Vite+React+TypeScript+Tailwind+shadcn/ui+Clerk setup.
    `npm run build` succeeds. TypeScript compiles clean. All dependencies installed.
  </done>
</task>

  <action>
    1. **Create `AuthGate.tsx`:**
       - Uses Clerk's `<SignedIn>` and `<SignedOut>` components
       - When signed out: render full-page centered Clerk `<SignIn />` component (per user decision: "full-page auth screen with centered form")
       - When signed in: render `{children}` (the dashboard app)
       - Style: center the sign-in form vertically and horizontally on a minimal background
       - Configure Clerk `<SignIn />` to show email/password, GitHub OAuth, and Google OAuth (these are configured in Clerk Dashboard, not in code — but ensure the appearance config doesn't restrict them)

    2. **Create 6 stub page components:**
       Each page is a simple functional component that renders a heading with the page name. These are placeholders for subsequent phases.
       - `OverviewPage.tsx` — "Overview" (default/home route)
       - `CostsPage.tsx` — "Costs"
       - `PoliciesPage.tsx` — "Policies"
       - `ApprovalsPage.tsx` — "Approvals"
       - `AlertsPage.tsx` — "Alerts"
       - `SettingsPage.tsx` — "Settings"

    3. **Configure routing in `App.tsx`:**
       - Use `react-router-dom` `BrowserRouter`, `Routes`, `Route`
       - Routes:
         - `/` → OverviewPage
         - `/costs` → CostsPage
         - `/policies` → PoliciesPage
         - `/approvals` → ApprovalsPage
         - `/alerts` → AlertsPage
         - `/settings` → SettingsPage
       - Wrap all routes inside `<AuthGate>` so nothing is accessible without signing in

    4. **Update `main.tsx`:**
       - Import and render `<App />` inside `<ClerkProvider>` and `<BrowserRouter>`
       - The component tree: `ClerkProvider > BrowserRouter > App`
       - App contains: `AuthGate > Routes`

    5. **Sign out support:**
       - For now, add a temporary sign-out button in the stub pages or App layout using Clerk's `<UserButton />` or `useClerk().signOut()`. This will be replaced by the sidebar user avatar in Plan 11-02. Just ensure sign-out works for verification.

    NOTE: Sign-in methods (GitHub, Google) are configured in the Clerk Dashboard, not in React code. The `<SignIn />` component automatically shows whatever methods the Clerk application has enabled. The `.env.example` documents the required key.
  </action>
