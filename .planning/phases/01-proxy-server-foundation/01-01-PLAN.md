<objective>
Set up the Govyn monorepo with TypeScript and build the core HTTP proxy server with versioned URL routing that transparently forwards requests to OpenAI, Anthropic, and custom endpoints.

Purpose: This is the foundational skeleton of the entire proxy product. Without this, nothing else can be built. The proxy must forward requests correctly to all three provider types using Node.js http module (not Express) for minimal latency overhead.

Output: A working HTTP proxy server that accepts requests on /v1/openai/*, /v1/anthropic/*, and /v1/custom/:name/* and forwards them to the correct upstream API, returning the response to the caller.
</objective>

  <action>
    Create src/router.ts — URL routing module:
    - Export `matchRoute(url: string, providers: Map)` that parses the request URL
    - Route patterns: /v1/openai/* -> openai provider, /v1/anthropic/* -> anthropic provider, /v1/custom/:name/* -> custom provider by name
    - Return RouteMatch with the resolved provider config and the upstream path (strip the /v1/openai prefix, keep the rest e.g. /v1/chat/completions)
    - Return null for unmatched routes
    - For OpenAI routes: strip /v1/openai prefix, forward remaining path (e.g., /v1/openai/v1/chat/completions -> /v1/chat/completions to api.openai.com)
    - For Anthropic routes: strip /v1/anthropic prefix, forward remaining path (e.g., /v1/anthropic/v1/messages -> /v1/messages to api.anthropic.com)
    - For Custom routes: strip /v1/custom/:name prefix, forward remaining path

    Create src/providers/openai.ts:
    - Export default provider config: baseUrl = https://api.openai.com, apiKeyEnv = OPENAI_API_KEY
    - Export function to map headers: read API key from env var, set Authorization: Bearer {key}, forward Content-Type and other relevant headers
    - Do NOT forward the host header (use upstream host)

    Create src/providers/anthropic.ts:
    - Export default provider config: baseUrl = https://api.anthropic.com, apiKeyEnv = ANTHROPIC_API_KEY
    - Export function to map headers: read API key from env var, set x-api-key: {key}, set anthropic-version header if not present, forward Content-Type
    - Do NOT forward the host header

    Create src/providers/custom.ts:
    - Export function to create custom provider config from name + config object
    - Forward API key from configured env var (if set), pass through all headers except host

    Create src/proxy.ts — request forwarding:
    - Export `forwardRequest(req: IncomingMessage, res: ServerResponse, routeMatch: RouteMatch)`
    - Use Node.js built-in http/https module (NOT node-fetch, NOT axios — keep it zero-dependency for speed per ADR-013)
    - Read the full request body from the incoming request
    - Create an outgoing request to routeMatch.provider.baseUrl + routeMatch.upstreamPath
    - Map headers using the provider's header mapping function
    - Forward the request body
    - Pipe the upstream response back to the client response (headers + status code + body)
    - For non-streaming responses: pipe directly
    - Measure and log the time from request start to first byte of upstream response (for latency tracking)
    - Handle connection errors gracefully: if upstream is unreachable, return 502 Bad Gateway with JSON error body

    Create src/server.ts — HTTP server:
    - Use Node.js `http.createServer()` (NOT Express, per BUILD_ROADMAP)
    - Accept config (port, host, providers map)
    - On each request: call matchRoute, if match found call forwardRequest, if no match return 404 with JSON error
    - Export `startServer(config: ProxyConfig)` that creates and starts the server
    - Log server start message with port

    Update src/index.ts:
    - Import startServer
    - Create a hardcoded default config for now (port 4000, host 0.0.0.0, default OpenAI + Anthropic providers)
    - Call startServer — YAML config loading comes in Plan 01-02
    - The hardcoded config is temporary; it will be replaced by YAML config in the next plan

    Create tests/router.test.ts:
    - Test that /v1/openai/v1/chat/completions matches openai provider with upstream path /v1/chat/completions
    - Test that /v1/anthropic/v1/messages matches anthropic provider with upstream path /v1/messages
    - Test that /v1/custom/my-llm/v1/chat/completions matches custom provider "my-llm" with upstream path /v1/chat/completions
    - Test that /unknown/path returns null (no match)
    - Test that /v1/custom/nonexistent/path returns null when provider not configured

    Create tests/proxy.test.ts:
    - Test that forwardRequest sends correct headers for OpenAI provider (Authorization: Bearer)
    - Test that forwardRequest sends correct headers for Anthropic provider (x-api-key)
    - Test that forwardRequest returns 502 when upstream is unreachable
    - Use a local http test server (create in test setup) to verify request forwarding without hitting real APIs
