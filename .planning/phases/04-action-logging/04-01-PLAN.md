**3. Create `src/action-logger.ts`:**

Implement `ActionLogger` class:

**Constructor:** `new ActionLogger(config: LoggingConfig)`. Creates log directory and payload subdirectory (`{directory}/payloads/`) using `fs.mkdirSync({recursive: true})`. Opens initial JSONL file for append writing. File naming: `govyn-{YYYY-MM-DD}.jsonl`. Initializes an internal write buffer (array of strings).

**`log(entry: LogEntry)` method:** The primary async log method. Serializes entry to JSON. If `config.stdout` is true, writes to `process.stdout.write()`. If `config.file` is true, pushes to internal buffer. Does NOT await file I/O — this is the zero-latency guarantee. Returns void synchronously.

**`storePayload(payloadId: string, requestBody: Buffer | null, responseBody: Buffer | null, truncated: boolean)` method:** Writes a JSON file to `{directory}/payloads/{payloadId}.json` containing `{ request_body, response_body, truncated, stored_at }`. Bodies are base64-encoded strings (handle binary). If body exceeds `maxBodySize`, truncate to limit and set `truncated: true`. This write is fully async (fs.promises.writeFile) — fire and forget with error logging to stderr.

**`flush()` method:** Writes buffered entries to the current JSONL file. Uses `fs.appendFileSync` on the buffered entries joined with newlines. Clears buffer. Called on interval (e.g., every 1 second via setInterval with unref()). Also callable manually.

**`getMode(agentId: string): LoggingMode`:** Returns agent-specific mode from `config.agentModes` or falls back to `config.defaultMode`.

**`setMode(agentId: string, mode: LoggingMode): void`:** Runtime mode toggle — updates the agentModes map. Does NOT persist to YAML.

**`getCurrentFilePath(): string`:** Returns path of current active JSONL file (useful for rotation).

**`close(): void`:** Flushes buffer, clears flush interval.

**ID generation:** Use `crypto.randomUUID()` for log entry IDs and payload IDs.

**1. Update `src/proxy.ts` — add ActionLogger to forwardRequest():**

Add optional `actionLogger?: ActionLogger` parameter to `forwardRequest()` signature (after `budgetEnforcer`, following the same optional dependency injection pattern).

Import `ActionLogger` and `LogEntry` types.

In both SSE and non-SSE `upstreamRes.on('end')` handlers, AFTER the existing cost recording block, add logging:

```typescript
if (actionLogger) {
  const mode = actionLogger.getMode(agentId);
  const logId = crypto.randomUUID();
  const payloadId = mode === 'full-payload' ? crypto.randomUUID() : null;

  const entry: LogEntry = {
    id: logId,
    timestamp: new Date().toISOString(),
    agent_id: agentId,
    provider: routeMatch.providerType,
    target: routeMatch.upstreamPath,
    model: usage?.model ?? null,
    input_tokens: usage?.inputTokens ?? null,
    output_tokens: usage?.outputTokens ?? null,
    cost: cost?.totalCost ?? null,
    priced: cost?.priced ?? false,
    latency_ms: Date.now() - requestStart,
    status: statusCode,
    has_payload: payloadId !== null,
    payload_id: payloadId,
  };

  actionLogger.log(entry);

  if (payloadId) {
    // Store payload async (fire and forget)
    const maxSize = actionLogger.config.maxBodySize;
    const reqBody = body; // already read earlier in forwardRequest
    const resBody = Buffer.concat(responseBodyChunks); // for non-SSE
    // For SSE: join sseChunks into a Buffer
    const truncated = (reqBody.length > maxSize) || (resBody.length > maxSize);
    actionLogger.storePayload(payloadId, reqBody, resBody, truncated);
  }
}
```

Note: `usage` and `cost` variables are already computed in the existing cost recording block. Make sure the logging code accesses them correctly (they may be scoped inside the existing `if (usage)` block — extract them to the outer scope so logging can access even if usage is null).

Also handle the proxy-error cases (upstream connection error, timeout) and the loop-detected case — log those too with status 502/429 and no token/cost info.

For the loop-detected early return, add a log entry before `sendLoopDetectedError()` with status 429, cost null, model null, has_payload false.
