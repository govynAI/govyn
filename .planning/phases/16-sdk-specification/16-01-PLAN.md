- OpenAI: `base_url = {GOVYN_PROXY_URL}/v1/openai`
  - SDK appends paths like `/chat/completions`, producing `{proxy}/v1/openai/v1/chat/completions`
  - Proxy strips `/v1/openai` prefix, forwards `/v1/chat/completions` to `https://api.openai.com`
- Anthropic: `base_url = {GOVYN_PROXY_URL}/v1/anthropic`
  - SDK appends paths like `/v1/messages`, producing `{proxy}/v1/anthropic/v1/messages`
  - Proxy strips `/v1/anthropic` prefix, forwards `/v1/messages` to `https://api.anthropic.com`

- Include warning about double-slash bug: ensure no `//` in constructed URLs

**4. Header Injection**
- SDKs MUST set `X-Govyn-Agent: {agent_id}` as a default header on every request
- Header is case-insensitive (Node.js HTTP parser lowercases to `x-govyn-agent`)
- When `X-Govyn-Agent` is present, it takes priority over API-key-based identification

**5. API Key Convention (SPEC-02)**
Two modes, clearly documented with guidance:

**Passthrough mode (recommended default):**
- SDK sends `Authorization: Bearer govyn-passthrough` (OpenAI) or `x-api-key: govyn-passthrough` (Anthropic)
- Proxy strips placeholder, injects real API key from env var (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`)
- Use when: proxy holds all real API keys (recommended setup)

**Key-storage mode (scoped keys):**
- Operator generates scoped keys like `gvn_ra_xxxx` per agent
- SDK sends the scoped key in the auth header
- Proxy uses this to identify the agent AND injects real upstream API key
- Use when: agent identity must be cryptographically tied to specific keys
- Note: when `X-Govyn-Agent` header is also present, it takes priority for identification

**6. Constructor Requirements**
- `agent_id`: MANDATORY. Resolution order: constructor arg > `GOVYN_AGENT_ID` env var > raise error. MUST NOT silently default to "unknown". This is a locked decision.
- `proxy_url`: Resolution order: constructor arg > `GOVYN_PROXY_URL` env var > default `http://localhost:4000`
- `api_key`: Resolution order: constructor arg > default `"govyn-passthrough"`
- `max_retries`: MUST be set to `0`. Proxy counts each request independently. SDK retries cause double-billing and incorrect loop detection. This is a locked decision.
- `base_url`: Computed from proxy_url + provider route prefix. MUST NOT be user-configurable (it's derived).
- `default_headers`: MUST include `X-Govyn-Agent: {agent_id}`
- Include pseudocode constructor example from research showing the resolution chain

**8. Health Check**
`GET /health` returns:
```json
{
  "status": "ok",
  "version": "0.0.1",
  "uptime_seconds": 123
}
```
SDKs MUST implement `check_proxy()` (Python) / `checkProxy()` (Node.js) that:
1. Sends GET to `{proxy_url}/health`
2. Returns success if HTTP 200 and `status == "ok"`
3. Returns failure (or raises) otherwise
4. Has a configurable timeout (default: 5 seconds)

**9. Behavioral Rules**
- **No SDK-level retries:** `max_retries=0`. Rationale: proxy counts each request independently; retries cause double-billing and loop detection false positives.
- **Streaming passthrough:** SDKs MUST NOT buffer streaming responses. The upstream SDK's streaming behavior must be preserved unchanged. The proxy handles SSE passthrough transparently.
- **No URL manipulation:** SDKs MUST NOT modify, normalize, or strip path components from the base_url. The exact string must be passed to the upstream SDK constructor.
- **No silent fallbacks:** If the proxy is unreachable, let the upstream SDK's connection error propagate. Do NOT fall back to direct API access. The proxy IS the governance layer.
- **Header preservation:** SDKs MUST NOT remove or modify headers set by the proxy on responses (e.g., `X-Govyn-Budget-Warning`, `retry-after`).

| Budget error | `GovynBudgetExceededError` | `GovynBudgetExceededError` |

| Health check | `check_proxy()` | `checkProxy()` |
| Agent ID param | `agent_id` | `agentId` |
| Proxy URL param | `proxy_url` | `proxyUrl` |

Follow each language's naming convention (snake_case for Python, camelCase for Node.js) but keep class names identical across languages for brand consistency.
